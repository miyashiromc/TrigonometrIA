Prompt para Generar una Nueva Paleta de Colores Accesible
Rol: Actúa como un desarrollador front-end y diseñador UI/UX con experiencia en accesibilidad web (WCAG).

Contexto: Te proporciono el código fuente de una aplicación de tutoría de trigonometría construida con React, TypeScript y Tailwind CSS. La aplicación tiene un tema claro y uno oscuro. La paleta de colores actual (brand-primary, brand-secondary, base, dark-base, etc.) necesita ser renovada para mejorar la legibilidad del texto, el contraste entre los elementos y la comodidad visual para los usuarios, especialmente aquellos con discapacidades visuales.

Objetivo Principal:
Generar una nueva paleta de colores cohesiva y accesible para la aplicación, y proveer instrucciones claras sobre cómo implementarla en los archivos .tsx existentes. La nueva paleta debe ser moderna, atractiva y cumplir con los estándares de accesibilidad WCAG AA, apuntando a AAA siempre que sea posible.

Tarea Detallada:

Analiza los Archivos: Revisa los archivos .tsx proporcionados para identificar todos los colores utilizados, tanto los definidos por el tema (ej. bg-brand-primary, text-base-content) como los colores de estado codificados directamente (ej. bg-green-500, text-red-700).

Define una Nueva Paleta de Colores: Propón una nueva paleta de colores completa para los modos claro y oscuro. La paleta debe incluir:

Color Primario (Brand Primary): Un color principal para la marca, usado en botones, enlaces y elementos activos. Debe tener excelente contraste sobre los fondos base.

Color Secundario (Brand Secondary): Un color de acento para elementos secundarios o para indicar estados "hover".

Colores Base (Fondo): Define una gama de colores para los fondos (base-100, base-200, base-300) tanto para el tema claro como para el oscuro. Deben ser colores suaves y que no cansen la vista.

Colores de Contenido (Texto): Define los colores para el texto (base-content, dark-base-content) que aseguren un alto contraste sobre los colores de fondo correspondientes.

Colores de Estado:

Éxito (Success): Un color verde accesible para notificaciones de éxito y respuestas correctas.

Error (Error): Un color rojo accesible para mensajes de error y respuestas incorrectas.

Información (Info/Warning): Colores azul o amarillo para otros tipos de notificaciones.

Proporciona Guía de Implementación:

Muestra cómo se vería la configuración de colores en un archivo de configuración de Tailwind CSS (tailwind.config.js), incluso si el archivo no fue proporcionado. Esto servirá como la "fuente de la verdad" para los colores.

Selecciona 3 archivos clave (por ejemplo, ChatView.tsx, PracticeView.tsx y LoginView.tsx) y muestra los cambios específicos de clases de Tailwind CSS necesarios para aplicar la nueva paleta. Por ejemplo: cambiar bg-brand-primary por el nuevo color o clase correspondiente.

Ejemplo de Paleta Sugerida (puedes basarte en esta o proponer una mejor):

Tema Claro:

primary: Un azul índigo intenso (ej. #4338CA).

secondary: Un azul más claro (ej. #818CF8).

background: Blanco hueso / Gris muy claro (ej. #F9FAFB).

text: Casi negro (ej. #111827).

success: Verde oscuro (ej. #059669).

error: Rojo oscuro (ej. #B91C1C).

Tema Oscuro:

primary: Un azul lavanda brillante (ej. #A5B4FC).

secondary: Un azul más sutil (ej. #6366F1).

background: Azul marino muy oscuro / casi negro (ej. #1F2937).

text: Gris muy claro (ej. #D1D5DB).

success: Verde claro (ej. #34D399).

error: Rojo claro (ej. #F87171).

Formato de Respuesta:
La respuesta debe estar estructurada en tres secciones claras:

Nueva Paleta de Colores Propuesta: Una tabla o lista que defina los nuevos colores para los temas claro y oscuro.

Configuración para tailwind.config.js: Un bloque de código con la configuración sugerida.

Ejemplos de Implementación: Bloques de código mostrando el "antes y después" o los cambios necesarios en los 3 archivos .tsx seleccionados.
